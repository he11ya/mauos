# MAUOS — контекст для агента (Claude Code / Codex)

Этот файл в репозитории **обезличенный**: без имён, без путей к твоему диску и без личных ритуалов.

## Роль агента

Помогать с Markdown, структурой vault, скриптами и Git **внутри репозитория MAUOS** и связанного Obsidian‑слоя. Соблюдать принципы из `README.md` (file‑first, воспроизводимость).

## Язык

По умолчанию отвечай на **русском**, если пользователь не попросил иначе.

## Осторожность

- Не выдумывай содержимое закрытых или отсутствующих файлов.
- **Секреты** не коммитить: ключи API, токены, пароли — только вне Git (см. `.env.example`, `.gitignore`).
- Не предлагай опасные команды вроде слепого `git add . && commit && push` без явного запроса пользователя.

## Что НЕ коммитить в публичный GitHub

Репозиторий **публичный**. Следующие файлы никогда не должны попадать в коммиты:

**Операционные файлы (покрыты `.gitignore`):**
- `**/logs/` — session-логи содержат локальные пути, личные детали рабочего процесса
- `**/NEXT_SESSION.md` — содержит абсолютные пути к файлам на диске пользователя
- `**/scratch/`, `**/private/` — черновики и личные заметки

**Любые файлы, содержащие:**
- Абсолютные пути вида `/Users/...` или `~/...`
- Имена, email-адреса, контакты людей
- Внутренние детали проектов, не предназначенные для публики
- Таблицы участников курсов

**Что безопасно коммитить:**
- Лекции, слайды, домашки, воркшопы (публичный образовательный контент)
- README и документация проектов
- CLAUDE.md (обезличенный, без личных данных)
- Скрипты без хардкоженных путей
- `.gitignore`, `.env.example`

**Правило перед `git add`:** проверить, нет ли в файле путей `/Users/...`, имён, email или других личных данных.

## Личные настройки

Если нужен свой тон, персона или локальные пути — создай **`CLAUDE.local.md`** в корне репозитория (файл в `.gitignore`, в Git не попадёт).
